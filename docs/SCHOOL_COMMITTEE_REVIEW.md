# BOGUNON 학교운영위원회 검토 자료

작성일: 2026-07-19

서비스 주소: https://bogunon.vercel.app/

운영자: BOGUNON 운영자 쑤캥T

연락처: sungandi@sen.go.kr

## 1. 도입 목적

BOGUNON은 교사가 자신의 일정, 할 일, 업무 절차와 일반적인 운동 기록을 정리하는 개인 생산성 도구다. 학생을 관리하거나 학생 건강기록을 작성하는 시스템이 아니다.

## 2. 학생 개인정보 처리 여부

| 확인 항목 | 답변 | 운영 원칙 |
| --- | --- | --- |
| 학생 개인정보를 다루는가 | 아니오 | 학생 이름, 학번, 연락처 등 입력 금지 |
| 학생 건강·민감정보를 다루는가 | 아니오 | 질병명, 상담·진료 내용 등 입력 금지 |
| 학생 정보를 외부 서비스에 전송하는가 | 아니오 | AI 요청 전 민감정보 형태 차단 |

학생용 계정을 제공하지 않으며 교사 개인 계정만 사용한다. 로그인 화면, 이용약관, 개인정보처리방침에 입력 금지 원칙을 공개한다.

## 3. 처리 정보와 목적

| 구분 | 정보 | 목적 |
| --- | --- | --- |
| 계정 | 이메일, Google 계정 식별자, 인증·세션 정보 | 로그인과 계정 보안 |
| 사용자 작성 정보 | 일정, 할 일, 워크플로, 체크리스트, 링크·알림, 일반 운동 기록, 설정 | 개인 업무 관리와 동기화 |
| 선택적 AI | 최대 1,200자 요청문, 사용자가 선택한 업무 맥락 | 업무 정리 제안 생성 |
| 접속 정보 | IP, 사용자 에이전트, 요청 시각·경로, 오류 기록 | 보안 대응과 장애 분석 |

## 4. 외부 서비스

| 서비스 | 역할 | 학생 정보 전송 |
| --- | --- | --- |
| Supabase | Google 인증, 계정별 데이터베이스 | 없음 |
| Vercel | 웹 호스팅, CDN, 운영 로그 | 없음 |
| OpenAI | 사용자가 실행한 선택적 AI 제안 | 없음. 학생 정보 형태는 전송 전 차단 |
| Google | OAuth 로그인 | 없음 |

국외 이전 국가, 항목, 시기, 보유 기간과 거부 방법은 공개 개인정보처리방침의 `처리 위탁과 국외 이전` 절에 명시한다.

## 5. 안전성 조치

- HTTPS와 보안 쿠키를 사용한다.
- Supabase 행 수준 보안(RLS)으로 계정별 접근을 분리한다.
- 콘텐츠 보안 정책(CSP), 클릭재킹 차단, MIME 스니핑 차단, 리퍼러·브라우저 기능 제한 헤더를 적용한다.
- AI 기능은 필요한 제목 중심의 최소 맥락만 전송하고, 학생 개인정보·건강정보 형태를 코드에서 차단한다.
- 사용자가 AI 기록 저장을 선택하지 않으면 앱 데이터베이스에 요청·결과를 저장하지 않는다.
- 공개 개인정보처리방침에서 열람·정정·삭제·처리정지·탈퇴 방법과 보호책임자 연락처를 안내한다.

## 6. 위원회 확인 체크리스트

- [ ] 이용 목적이 교사 개인 업무 관리에 한정되는지 확인
- [ ] 학생 개인정보와 건강정보를 입력하지 않는 운영 지침 승인
- [ ] 외부 서비스와 국외 이전 고지 확인
- [ ] 계정 발급 대상과 퇴직·업무 변경 시 삭제 절차 확인
- [ ] 보안 사고 발생 시 운영자 연락과 학교 내부 보고 절차 확인
- [ ] 공개 개인정보처리방침과 이용약관 확인

## 7. 결론 요청안

BOGUNON을 학생 개인정보·민감정보를 입력하지 않는 교사 개인 업무 관리 도구로 사용하며, 공개된 개인정보처리방침과 본 운영 원칙을 준수하는 조건으로 도입을 검토한다.
