# Privacy

## 1. 적용 범위

보건온은 보건교사의 업무와 일정 운영 도구이며 학생 개인의 건강기록 시스템이 아니다. 이 원칙은 일반 입력, Workflow, 검색, 로그와 Phase 7 AI 기능에 동일하게 적용한다. 사용자는 학생을 식별하거나 건강 상태를 드러내는 정보를 입력하지 않는다.

## 2. 입력·저장·전송 금지 정보

- 학생 이름, 학번, 생년월일, 개인을 식별할 수 있는 반·번호 조합
- 학생·보호자 연락처와 계정 정보
- 질병명, 증상, 진단, 검진 결과, 투약·처치·상담·병원 이용 내용
- 장애·특수교육 정보, 개인을 추론할 수 있는 사건 설명
- 위 정보가 담긴 파일, 링크 본문, 캡처와 외부 문서 내용
- Google access token, Supabase session·secret·service role key와 AI provider API key

부분 마스킹, 이니셜과 가명도 다른 정보와 결합해 개인을 알아볼 수 있으면 허용하지 않는다.

## 3. 허용되는 표현

업무 수행에 필요한 대상은 범주와 비식별 수량으로 표현한다.

- `2학년 관련 담임 확인 필요`
- `후속 확인 3건`
- `미제출 5건`
- `외부 기관 회신 대기`

자유 입력, placeholder, 템플릿과 mock 데이터도 같은 규칙을 지킨다.

## 4. Phase 7 AI 개인정보 원칙

- AI 사용은 선택적이며 사용하지 않아도 기존 Task·Event·Workflow 기능을 모두 사용할 수 있어야 한다.
- 요청 전에 민감정보 금지 안내를 표시하고, 사용자가 현재 선택한 최소 문맥만 서버를 통해 provider에 전송한다.
- 전체 업무 목록, 전체 캘린더, 전체 Workflow와 사용자 프로필을 편의를 이유로 자동 첨부하지 않는다.
- 메모·설명은 기본 제외한다. 사용자가 명시적으로 포함한 경우에도 금지 정보 입력 책임과 전송 범위를 미리 안내한다.
- provider에는 인증 토큰, 이메일, `user_id`와 다른 사용자 데이터를 보내지 않는다. 선택된 업무 참조 ID는 복제·적용 대상을 식별하는 경우에만 최소 전송하며, 실제 적용 시 현재 사용자 RLS로 다시 검증한다.
- AI 입력·응답·미리보기는 기본적으로 저장하지 않는다. 선택적 기록은 `ai_preferences.history_enabled=false`가 기본이며, 설정과 해당 요청에서 사용자가 명시적으로 opt-in한 경우에만 사용자별 RLS가 적용된 request·draft 행을 생성한다.
- AI는 삭제 기능을 갖지 않으며 변경은 구조화된 미리보기와 사용자 확인 뒤 기존 권한 경로로만 반영한다.
- provider 결과와 사용자가 수정한 구조화 미리보기는 화면 표시·기록·적용 전에 동일한 민감정보 검사를 다시 통과해야 한다.

## 5. 저장과 접근 통제

- Supabase Database가 사용자 원본의 단일 저장소다. localStorage는 저장 전 폼 초안과 UI 상태만 허용한다.
- 모든 사용자 소유 테이블은 `user_id`를 가지며 인증 사용자 자신의 행만 접근하도록 RLS를 활성화한다.
- 하위 행은 부모 ID와 `user_id`를 함께 검증해 다른 사용자 부모에 연결할 수 없게 한다.
- 브라우저가 전달한 `user_id`를 신뢰하지 않으며 서버 인증 세션과 RLS를 최종 기준으로 사용한다.
- AI 확인 이후의 생성·수정도 기존 Repository/RPC와 RLS를 사용한다. provider 응답에 데이터베이스 권한을 부여하지 않는다.

## 6. 로그와 오류

제목, 메모, 체크리스트, 링크, AI 입력·응답과 구조화된 제안 원문을 console, 서버 로그, 오류 추적 또는 분석 도구에 기록하지 않는다. 로그에는 비식별 request ID, 처리 시간, provider 종류, 상태·오류 분류와 제한 카운터만 허용한다. 사용자 화면의 오류에도 입력 원문, 다른 사용자의 데이터, provider 응답과 비밀값을 포함하지 않는다.

## 7. 보존과 사용자 제어

- 기본 AI 요청은 응답 반환 후 애플리케이션에서 보존하지 않는다.
- 선택적 기록은 기본 off, 명시적 동의, 목적과 보존 기간 안내, 사용자별 조회·삭제, 로그 비수집과 RLS 테스트를 유지한다. opt-out 이후에는 신규 기록을 즉시 중단한다.
- 사용자가 AI 미리보기를 취소하면 제안은 저장되지 않는다.
- 로그아웃 시 저장 전 폼 초안과 메모리의 AI 화면 상태를 제거한다.

## 8. 배포 전 확인

- 실제 provider의 데이터 이용·보존 설정과 계약이 제품 정책에 맞는가?
- 서버 전용 환경 변수가 브라우저와 로그에 노출되지 않는가?
- AI payload가 선택한 최소 필드만 포함하는가?
- 민감정보를 유도하는 문구·fixture·screenshot이 없는가?
- history off에서 DB와 localStorage에 AI 대화가 남지 않는가?
- 사용자 A/B와 비로그인 RLS, rate limit과 timeout 실패에서 데이터가 노출되거나 부분 저장되지 않는가?
